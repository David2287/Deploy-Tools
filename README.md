# Deploy Tool

**Версия:** 1.0.0  
**Python:** 3.8+  
**Платформа:** Windows (требуются права администратора)

Система развёртывания программного обеспечения для корпоративной среды Windows. Приложение с графическим интерфейсом (GUI) на Tkinter для удалённой установки ПО на рабочие станции через PowerShell и SMB.

## 🚀 Функционал

- **Проверка устройств** — Ping, разрешение DNS, проверка через Active Directory (опционально)
- **Валидация учётных данных** — Проверка админских прав через PowerShell
- **Удалённое развёртывание** — Двухэтапная установка: Сетевой ресурс → Админ ПК → Целевой ПК
- **Поддержка ОС** — Windows 7/Server 2008 и Windows 10/11
- **Журналирование** — Полный лог всех операций с экспортом
- **Темы оформления** — Светлая и тёмная тема
- **Проверка через AD** — Интеграция с LDAP (опционально)

### Процесс развёртывания (7 этапов)

1. **Проверка пакетов** — Проверка доступности установщиков на сетевом ресурсе
2. **Создание папки на админском ПК** — `C:\Temp\Deploy`
3. **Копирование на админский ПК** — Локальное копирование установщиков
4. **Создание папки на целевом ПК** — `C:\Temp\Install` через SMB
5. **Копирование на целевой ПК** — Передача файлов через `\\COMPUTER\C$\Temp\Install`
6. **Установка** — Выполнение установщиков через PowerShell Remoting
7. **Очистка** — Удаление временных файлов на обоих ПК

## 📦 Требования

### Системные требования
- **ОС:** Windows 10/11 или Windows Server 2016+
- **Права:** Администратор домена или локальный администратор
- **Python:** 3.8 или выше (только для запуска из исходников)
- **PowerShell:** 5.1 или выше
- **Сеть:** Доступ к целевым ПК через SMB (порт 445)

## 🔧 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/David2287/Deploy-Tools.git
cd Deploy-Tools
```
2. Создание виртуального окружения (рекомендуется)
```bash
python -m venv venv
venv\Scripts\activate
```
3. Установка зависимостей
```bash
pip install -r requirements.txt
```
▶️ Запуск из исходников

Вариант 1: Через run.bat (рекомендуется)
```bash
run.bat
```
Вариант 2: Напрямую через Python
```bash
python main.pyw
```

Важно: Приложение требует прав администратора. Если запустить без прав, появится окно с ошибкой.
Проверка структуры
Перед первым запуском проверьте структуру проекта:
```bash
python check_structure.py
```

## 🛠 Сборка в EXE
Автоматическая сборка (рекомендуется)

Запустите build.bat от имени администратора:
```bash
build.bat
```

Что делает скрипт:
Очищает старые сборки (build/, dist/)
Создаёт spec-файл PyInstaller с правильными настройками
Собирает EXE с включением всех модулей (gui, core, utils, config)
Проверяет результат
Результат: dist\DeployTool.exe

# Ручная сборка через PyInstaller

Если нужно настроить параметры вручную:
```bash
# Установка PyInstaller
pip install pyinstaller
```
# Сборка
```bash 
pyinstaller DeployTool.spec --clean
```

# Структура spec-файла

Spec-файл включает:
Hidden imports: Все модули проекта (gui, core, utils, config)

Data: Папки gui, core, utils, config как данные

Библиотеки: tkinter, ldap3, PIL

Настройки: Оконное приложение (без консоли), UPX сжатие

# ⚙️ Настройка
Настройка приложений (config/settings.py)
Перед использованием настройте пути к установщикам:
Python

# config/settings.py
```bash
APPLICATIONS = {
    "Google Chrome": {
        "Win10": r"\\server\share\apps\chrome\chrome_setup.exe",
        "Win7": r"\\server\share\apps\chrome\chrome_win7.exe",
        "args": "/silent /install",
        "timeout": 300
    },
    "Mozilla Firefox": {
        "Win10": r"\\server\share\apps\firefox\firefox_setup.exe",
        "Win7": r"\\server\share\apps\firefox\firefox_win7.exe", 
        "args": "/S",
        "timeout": 180
    },
    # Добавьте свои приложения
}
```

# Настройки PowerShell
```bash 
POWERSHELL_CONFIG = {
    "execution_policy": "Bypass",
    "timeout": 120
}
```

# Настройки Active Directory (опционально)
```bash 
AD_CONFIG = {
    "domain": "company.local",
    "server": "dc.company.local",
    "use_ssl": False,
    "port": 389,
    "timeout": 30
}
```

### Параметры приложений

```Table
Параметр	Описание	Пример
Win10	Путь к установщику для Windows 10/11	\\server\share\app.exe
Win7	Путь к установщику для Windows 7/2008	\\server\share\app_win7.exe
args	Аргументы тихой установки	/S, /silent, /quiet
timeout	Таймаут установки (секунды)	300
```

## 🖥 Использование

### 1. Запуск приложения
Запустите DeployTool.exe (или python main.pyw) от имени администратора
Приложение проверит права администратора при старте
### 2. Проверка устройства
Введите имя компьютера (например, WORKSTATION01 или WORKSTATION01.company.local)
Нажмите "Проверить доступность"
Дождитесь результатов:
✓ AD: Активен — Компьютер найден в Active Directory и включён
✓ Сеть: Доступен — Ping проходит, но AD не настроен
⚠ AD: Отключён — Компьютер в AD, но отключён
✗ Недоступен — Нет ни ping, ни записи в AD
### 3. Учётные данные
Введите логин в формате DOMAIN\username или username@domain.local
Введите пароль
Нажмите "Проверить учётные данные" для валидации
Учётные данные кэшируются на время сессии (если включена галочка)
### 4. Выбор программ
Отметьте нужные программы галочками
Используйте "Выбрать все" или "Сбросить" для массового выбора
Система автоматически определит ОС целевого ПК и выберет правильный установщик
### 5. Запуск установки
Нажмите "Запустить установку"
Подтвердите операцию
Следите за прогрессом в журнале выполнения
После завершения появится уведомление об успехе или ошибке
### 6. Экспорт логов
"Файл" → "Экспорт лога" — Сохранить лог в текстовый файл
Логи также сохраняются автоматически в папке logs/


## 🐛 Устранение неполадок
Приложение не запускается

```Table
Проблема	Решение
"Требуется запуск от имени администратора"	Нажмите ПКМ на файл → "Запуск от имени администратора"
"Ошибка импорта модулей"	Убедитесь, что все папки (gui, core, utils, config) содержат __init__.py
"Требуется Python 3.8 или выше"	Обновите Python до версии 3.8+
Ошибки сборки EXE
```
```Table
Проблема	Решение
pyinstaller не найден	Установите: pip install pyinstaller
Ошибка ModuleNotFoundError в EXE	Проверьте, что модуль указан в hiddenimports в DeployTool.spec
EXE не включает модули	Убедитесь, что datas в spec-файле содержит все папки проекта
Ошибки развёртывания
```
```Table
Проблема	Решение
"Не удалось создать папку на целевом ПК"	Проверьте доступ к административной шаре \\COMPUTER\C$
"Ошибка доступа"	Убедитесь, что учётные данные имеют права администратора на целевом ПК
"PowerShell execution policy"	На целевом ПК выполните: Set-ExecutionPolicy RemoteSigned -Scope LocalMachine
Установщик не запускается	Проверьте, что путь к установщику корректен и файл доступен
Таймаут установки	Увеличьте timeout в настройках приложения в config/settings.py
Проверка через AD не работает
Если не требуется интеграция с Active Directory, приложение будет работать без ldap3. Для включения:
```

```bash
pip install ldap3
```

И настройте AD_CONFIG в config/settings.py.
Логи и диагностика
Все операции логируются в:
GUI-консоль (в реальном времени)
Файл: logs\deploy_YYYYMMDD_HHMMSS.log
Для детальной диагностики проверьте:
Включен ли WinRM на целевом ПК: Test-WSMan COMPUTERNAME
Доступен ли SMB: Test-NetConnection -ComputerName COMPUTERNAME -Port 445
Правильность учётных данных: попробуйте подключиться вручную через PowerShell
